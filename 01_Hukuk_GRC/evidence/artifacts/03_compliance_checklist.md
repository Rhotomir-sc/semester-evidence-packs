# Compliance Checklist — IT Law & Cybersecurity (Mini Pack)

## A) KVKK — Personal Data Protection
- [ ] Purpose is defined and documented (purpose limitation)
  - Proof: short purpose statement + data inventory note
- [ ] Only necessary data is collected (data minimization)
  - Proof: required fields list + justification
- [ ] Legal basis / consent path is clear
  - Proof: consent log + notice text version
- [ ] Data subject rights handling exists (access/correction/deletion/objection)
  - Proof: 1-page request workflow

## B) Logs & Attribution
- [ ] Systems collect IP + port + timestamp where appropriate
  - Proof: sanitized log snippet + retention note

## C) Cybercrime risk awareness (TCK context)
- [ ] Unauthorized access & misuse risks are covered (policy + controls)
  - Proof: access control rules + monitoring note

## D) Digital Evidence mindset (CMK 134 concept)
- [ ] Evidence integrity is verifiable via hashing
  - Proof: “hash recorded” step in investigation note
- [ ] Chain-of-custody basics are documented
  - Proof: short template (who/when/what/hash)

## E) 5651 overview (if relevant)
- [ ] Takedown / access-blocking workflow exists
  - Proof: escalation steps + tracking table

## F) E-contracts & e-signature
- [ ] Click-wrap / consent evidence is stored (who/when/version)
  - Proof: consent log fields (userID, timestamp, version)

## G) Ads & unfair competition
- [ ] Marketing is consent-based; unsubscribe works
  - Proof: opt-in record + unsubscribe process

## H) IP / Industrial rights
- [ ] Licenses are respected (no unlicensed reuse)
  - Proof: license note for reused content
- [ ] Trademark/domain abuse awareness exists
  - Proof: reporting channel + takedown note
